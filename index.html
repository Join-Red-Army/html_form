<form>

Используется с двумя обязательными атрибутами: action и method.
Следует также указать name, чтобы отделять её от других форм.
Элементы внутри формы автоматически привязываются к ней.

<fieldset>

Визуально группирует несколько элементов формы. Рамка вокруг примера ниже - это fieldset.

<legend>

Подпись для form или fieldset
Legend!

<datalist>

Работает как т9: при вводе текста будет предлагать предзаданные варианты.
Внутри содержатся элементы option.
Используется совместно с input.

<select>

Представляет собой список option, по которому можно кликнуть и выбрать один или несколько (атрибут multiple) предзаданных вариантов.

<optgroup>

Группирует опции внутри select.

<progress>

Показывающий ход выполнения задачи.
Внутри атрибуты: max, value.
70%

<meter>

В зависимости от значения, цвет полосы будет меняться.
Внутри атрибуты: min, max, low, high, optimum и value.
85

<button>

Кнопки могут создаваться через теги input или button
Атрибуты type: button, submit, reset, которые:

<input>

Для получения информации от пользователя
Главный атрибут - type.
У type множество значений.

text

Ввод любой небольшой строки
Значение по умолчанию

password

Любые значения.
Поле защищено.

number, tel, email

Ну соответственно.

search

Поле для поисковых запросов.
Браузеры могут прикрутить не стандартное свойство onSearch.

radio

Выбрать только один вариант из предложенных.
Используется совмстно с label.


checkbox

Выбрать только несколько вариантов из предложенных.
Используется совместно с label.


submit, reset, image,

Кнопки для отправки и сброса формы.
Кнопки могут создаваться через теги input или button

file

Выбрать и прикрепить к форме файл

range

Такой вот бегунок.
Внутри устанавливаются значения min, max, value, step.
